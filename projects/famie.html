FAMIE PARENTAL CONTROL SYSTEM

Research, Full-Stack Development & Implementation

"Optimizing Screen Time Management For Children's Devices: Leveraging Token Bucket And Time-Based Algorithms In A Famie Parental Control System" is a computer science research study and software solution engineered to address modern screen-time dependencies and digital safety.

The comprehensive study consists of three distinct systems working in unison: a Parent Application, a Child Companion Application, and a Centralized Web Administration panel. On this project, I was assigned to build the entire full-stack architecture of both the Famie (Parent) and FamKid (Child) mobile applications—handling everything from the cross-platform user interfaces to the backend REST API workflows and database routing.

The Three Core Systems
  • Famie (Parent App): The mobile controller built for parents to manage their children's digital habits. It provides an intuitive, real-time command center where parents can easily link a child's device by scanning a generated QR code, configure individual profiles, and set specific time schedules to control exactly when kids are allowed to use or not use their devices.
  • FamKid (Child App): The companion mobile application installed on the child's device. Upon setup, it generates a unique QR code containing the device name and Android ID to securely identify the child's phone. Once the allowed time schedule begins, the background monitoring algorithm starts executing. As soon as the permitted time ends, the device is automatically locked. It can only be unlocked when a new allowed time schedule begins or if a parent manually overrides the lock using their custom set PIN code.
  • Admin Dashboard: The centralized web management console. This platform provides an administrative overview, tracking overall registration telemetry, monitoring global system logs, and providing dynamic configuration management over application themes.
Core Algorithms & How They Work

To ensure accurate, robust, and automated time enforcement without relying on heavy continuous server tracking, the system architecture operates on a dual-algorithmic engine:

  1. Token Bucket Algorithm (Real-Time Time Depletion): This algorithm treats screen time as a collection of virtual "tokens," where 1 token equals 1 second of permitted usage. When a child opens a configured schedule or device block, the FamKid application checks the bucket capacity. As long as tokens are present, screen access is granted, and tokens are depleted continuously second-by-second while the screen remains active. If a parent dynamically grants bonus minutes via the Famie app, the backend injects tokens immediately into the bucket. Once the bucket runs completely out of tokens (reaches 0), the application triggers a device restriction overlay to stop usage.
  2. Time-Based Algorithm (Automated System Reset): To maintain standard daily schedules, the system implements a time-based scheduling engine managed on the server side. Utilizing automated cron-job handlers within the Node.js backend environment, the system executes global updates at exactly 12:00 AM (Midnight) every day. This automated sweep refreshes the document matrices in the database, resets the active day logs, and replenishes the children's daily token buckets back to their baseline parent-configured allowances for the new day.

The complete full-stack environment backing the Famie and FamKid apps consists of a Node.js and Express.js REST API layer coupled with MongoDB Atlas for handling non-relational document data structures, cloud-hosted and deployed via Heroku. Code compilation was completed inside VS Code using GitHub for repository control, with feature optimization accelerated through AI debugging models.
